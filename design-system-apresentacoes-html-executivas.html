01 · Fundamentos

A aparência nasce de regras consistentes, não de decoração.

Este padrão prioriza clareza para decisão, hierarquia visual e portabilidade. Uma apresentação é uma sequência de mensagens, não uma página web rolável.

01

Mensagem antes do layout

Um slide comunica uma conclusão. Títulos devem dizer o que o público precisa entender.

02

Dados com leitura

KPI ou gráfico sempre acompanhado do insight e da unidade/período quando existirem dados reais.

03

Portabilidade total

Um único HTML, com CSS e JavaScript internos; abre localmente e imprime sem internet.

02 · Contrato do agente

Regras que toda apresentação gerada deve cumprir.

Obrigatório

Produção de deck profissional

Use esta página como padrão visual principal. Não invente números. Não gere aparência genérica de landing page.

01Entregar um único arquivo HTML autocontido, sem CDN, sem fonte remota e sem build.
02Usar palco fixo 16:9; cada slide deve caber na viewport sem rolagem interna.
03Adotar tokens, tipografia, componentes e exemplos visuais definidos nesta página.
04Começar pela narrativa executiva: problema, evidência, solução, impacto, decisão e próximos passos.
05Incluir navegação por teclado, progresso discreto, modo de impressão e movimento reduzido.
06Quando faltar dado real, sinalizar [INSERIR DADO] em vez de inventar conteúdo.

04 · Estrutura narrativa

Toda apresentação conduz a uma decisão.

O agente deve selecionar apenas os slides necessários para o objetivo, sem preencher páginas por obrigação.

A

Contexto e urgência

Capa, mensagem executiva e problema. Por que o tema importa agora?

B

Evidência e caminho

KPIs, causas, comparação atual versus futuro, fluxo ou arquitetura.

C

Decisão e execução

Ganho esperado, riscos, roadmap, decisão requerida e próximos passos.

06 · Dados e componentes

Escolha a forma visual pelo tipo de pergunta.

≡

Comparação

Barras horizontais, colunas agrupadas ou ranking para comparar segmentos, áreas ou cenários.

↗

Tendência

Linha ou colunas por período para evolução temporal; sempre com unidade e janela analisada.

◔

Composição

Barras empilhadas; donut apenas com poucas categorias e leitura evidente.

07 · Direção criativa

Profissional é sóbrio, preciso e intencional.

Faça

✓Títulos que já tragam a conclusão do slide.
✓Espaçamento amplo, alinhamento e poucos elementos fortes.
✓Gráficos simples acompanhados por insight.
✓Indicadores com contexto, período ou sinal de validação.
✓Slides escuros apenas para abertura ou destaque estratégico.

Evite

×Páginas cheias de parágrafos ou aparência de relatório.
×Gradiente roxo/neon e efeitos que pareçam template genérico.
×Tabelas longas, excesso de bordas e ícones decorativos.
×Números inventados para deixar o gráfico “completo”.
×Animações indispensáveis para entender a mensagem.
08 · Contrato técnico

O HTML final precisa funcionar sozinho.

O agente deve gerar slides com navegação local e impressão consistente, mesmo sem rede ou extensões.

`` e idioma correto.
CSS e JS dentro do arquivo.
Slides em `
`.
Palco fixo 16:9 escalonado.
Setas, espaço, Home e End.
Contador e barra discretos.
`@media print` + `@page` paisagem.
`prefers-reduced-motion` respeitado.
Sem CDNs, imports ou imagens remotas.
Sem corte ou rolagem no slide.
<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <style>
    :root { /* tokens desta página */ }
    .stage { aspect-ratio: 16 / 9; }
    .slide { width: 1600px; height: 900px; }
    @media print {
      @page { size: landscape; margin: 0; }
      .slide { page-break-after: always; }
      .controls { display: none; }
    }
    @media (prefers-reduced-motion: reduce) {
      * { animation: none !important; }
    }
  </style>
</head>
<body>
  <main class="deck">
    <section class="slide">...</section>
  </main>
  <script>/* navegação local */</script>
</body>
</html>
09 · Checklist final

O agente deve auditar o deck antes de entregar.

Existe uma mensagem central explícita?
Cada título expressa uma conclusão?
A sequência conduz a decisão ou próximo passo?
Cada slide cabe na tela sem rolagem interna?
O contraste é forte e a tipografia é legível?
Os gráficos têm insight, unidade e período?
Valores não fornecidos estão sinalizados?
Tokens, margens e componentes estão consistentes?
O arquivo abre localmente sem dependências?
A impressão gera um slide por página?
10 · Modelo de solicitação

Como pedir uma apresentação ao agente.

Crie uma apresentação executiva HTML usando obrigatoriamente o Design System de Apresentações HTML Executivas disponível em sua fonte pública.

Tema: [TEMA]
Público: [GERÊNCIA / DIRETORIA / EQUIPE TÉCNICA]
Objetivo da reunião: [DECISÃO OU ALINHAMENTO DESEJADO]
Mensagem central: [CONCLUSÃO PRINCIPAL]
Conteúdo disponível: [COLAR TEXTO, INDICADORES OU CONTEXTO]
Quantidade desejada: [N] slides.

Entregue primeiro a estrutura proposta e depois um único HTML autocontido.
Não use dependências externas e não invente dados ausentes.
