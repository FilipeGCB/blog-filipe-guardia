BASE_SLIDE_16_9_V1

Base determinística para página 16:9

Todo slide deve partir de um canvas fixo e de uma área útil explícita. Não posicione conteúdo em relação ao navegador; posicione em relação ao slide.

<section class="slide" data-title="Título do slide">
  <header class="slide-header">...</header>
  <main class="slide-content">...</main>
  <footer class="slide-footer">...</footer>
</section>

<style>
.slide{
  position:relative;
  width:1280px;
  height:720px;
  overflow:hidden;
  background:#fff;
}
.slide-content{
  position:absolute;
  inset:116px 64px 54px 64px;
  display:grid;
  gap:20px;
  min-width:0;
  min-height:0;
}
</style>
Obrigatório
Área útil explícita, alinhamentos comuns e espaçamento previsível.
Proibido
Elementos soltos com coordenadas independentes sem sistema de grid.

TEXT_CONTAINMENT_VISUAL_V2

Conter texto visualmente sem validar sua semântica

Nesta etapa, o sistema verifica apenas se o texto está legível, contido e bem distribuído. Ortografia, fatos, números e redação já devem ter sido aprovados antes.

function fitTextVisual(el, options = {}) {
  const min = options.minFontPx ?? 16;
  const max = options.maxFontPx ?? parseFloat(getComputedStyle(el).fontSize);
  let size = max;
  el.style.fontSize = `${size}px`;
  el.style.overflow = "hidden";
  el.style.overflowWrap = "anywhere";

  while (
    size > min &&
    (el.scrollHeight > el.clientHeight + 1 ||
     el.scrollWidth > el.clientWidth + 1)
  ) {
    size -= 0.5;
    el.style.fontSize = `${size}px`;
  }

  const failed =
    el.scrollHeight > el.clientHeight + 1 ||
    el.scrollWidth > el.clientWidth + 1;

  el.dataset.textFit = failed ? "fail" : "ok";
  return !failed;
}
Verificar
Overflow, corte, linha órfã, título espremido, contraste e tamanho mínimo.
Não verificar aqui
Se o texto está correto, completo, verdadeiro ou bem redigido.
Se falhar
Redimensionar a caixa, mudar composição ou dividir a página. Nunca reduzir indefinidamente.
FLOW_SPEC_ELK_SVG_V2

LLM define lógica; ELK posiciona; SVG Vivo desenha

Mermaid é adequado para validar a lógica. No HTML final, o fluxo aprovado deve ser convertido em especificação estruturada. O modelo não calcula coordenadas nem desenha conectores manualmente.

<script type="application/json" class="vivo-flow-spec">
{
  "id":"fluxo-demanda",
  "direction":"RIGHT",
  "availableWidth":1120,
  "availableHeight":470,
  "nodes":[
    {"id":"n1","type":"step","number":"01","title":"Registrar demanda","subtitle":"Entrada unificada","phase":"entrada"},
    {"id":"n2","type":"decision","title":"Já existe?","phase":"triagem"},
    {"id":"n3","type":"step","title":"Vincular e comunicar","phase":"triagem"},
    {"id":"n4","type":"step","title":"Analisar dados","phase":"analise"}
  ],
  "edges":[
    {"from":"n1","to":"n2"},
    {"from":"n2","to":"n3","label":"Sim"},
    {"from":"n2","to":"n4","label":"Não"}
  ]
}
</script>
// Responsabilidades do pipeline
// 1. LLM: produzir nodes, edges, tipos, fases e rótulos.
// 2. ELK: calcular x, y, largura, altura e rotas ortogonais.
// 3. SVG renderer: aplicar cards, tipografia, cores, sombras e marcadores.
// 4. Playwright: verificar o resultado renderizado.
// 5. LLM: criticar screenshot e corrigir a especificação ou composição.
Não usar: divs absolutas + linhas CSS + setas soltas + coordenadas inventadas pelo modelo.
CONNECTOR_INTEGRITY_V1

Integridade visual de setas e conectores

  • A seta deve iniciar e terminar em portas reais do nó, nunca no vazio.
  • Conector não atravessa texto, ícone, número de etapa ou rótulo.
  • Rótulos “Sim” e “Não” ficam próximos do primeiro segmento da ramificação.
  • Loops retornam pela periferia do diagrama e deixam evidente o ponto de retorno.
  • Cruzamentos são bloqueadores; se inevitáveis, a página precisa ser recomposta ou dividida.
  • Espessura, cor, curvatura e ponta da seta devem ser consistentes.

PAGE_FIT_GATE_V2

Dividir antes de comprimir

O gate de página ocorre antes do desenho final. O objetivo não é “fazer caber”; é preservar leitura e hierarquia.

Fluxo linear
Até 6 macroetapas por página.
Fluxo detalhado
Até 8 nós e 2 decisões como referência.
Ramificações
Até 4 destinos visíveis sem perda de leitura.
Falha
data-flow-fit="fail" exige nova paginação.

Listas, critérios, inventários e classificações devem virar cards, matriz, tabela ou legenda; não devem ser conectados como se fossem etapas.

PLAYWRIGHT_VISUAL_LOOP_V2

Loop visual obrigatório: renderizar → criticar → corrigir → repetir

Playwright não é apenas um teste final. É parte do processo de design. O HTML não pode ser chamado de final enquanto o agente não conseguir visualizar todos os slides e concluir uma revisão visual limpa.

LOOP:
  1. Abrir o HTML no Chromium.
  2. Aguardar fontes, scripts, ELK, SVG e animações estabilizarem.
  3. Capturar screenshot de cada slide em 1280×720.
  4. Comparar screenshots com o contrato visual aprovado.
  5. Criticar somente o visual:
     - composição, hierarquia, alinhamento e espaçamento;
     - caixas, setas, rótulos, loops e continuidade;
     - conteúdo visual presente e conectado conforme a estrutura aprovada;
     - cortes, overflow, sobreposição, contraste e legibilidade;
     - equilíbrio do canvas e consistência entre slides.
  6. Registrar defeitos objetivos.
  7. Corrigir HTML, CSS, SVG ou flow-spec.
  8. Reabrir e repetir.
  9. Encerrar apenas após 2 passagens consecutivas sem bloqueadores visuais.
Texto: validação visual
Dentro da caixa, legível, sem corte e com boa quebra.
Texto: fora desta etapa
Fatos, ortografia, números, completude e redação.
Playwright indisponível
Não liberar como final. Marcar rascunho_sem_visualizacao.

PROHIBITED_PATTERNS_V1

Padrões proibidos

  • Desenhar fluxograma complexo com caixas e setas posicionadas manualmente.
  • Reduzir fonte indefinidamente para evitar nova página.
  • Usar PNG/Base64 do Mermaid como visual final.
  • Declarar “final” sem conseguir abrir e visualizar cada slide.
  • Fazer apenas checagem de DOM e considerar o design aprovado.
  • Validar conteúdo textual dentro do loop visual e misturar tipos de QA.
  • Ignorar regressões provocadas por uma correção; toda alteração exige nova captura.
